2. 与 1goplus 的差异（V1 精简）

说明：差异主要是 V1 范围裁剪，不代表视觉或交互标准降低。

2.1 模块级差异

模块 与 1goplus 的差异点 V1 处理
M1 登录/注册 邮箱登录不保留；邀请码不做 移除/延后
M2 Home Jackpot 三卡区不做 整块隐藏
M3 签到 Quest 页面不做 仅保留签到弹窗
M4 coins coin 消耗仅保留 Betting 其余不做
M5 游戏接入 Demo、Recent Play 不做；多分类里仅 Slots 可玩 入口可见，点击 Toast 提示
M6 个人中心 Gift Code、帮助中心、通知设置不做 仅保留核心项与 Change Password

2.2 全局 V1 不做（V2+）

  • Deposit / Withdraw（V2）
  • VIP System（V3）
  • Lucky Spin（V2）
  • Leaderboard（V2）
  • Affiliate（V3）
  • Multi-game Types（V2）
  • Gift Code 兑换（V2）
  • Email Login（V2）
  • Invite Code（V2）

2.3 品牌差异

  • 品牌文案可使用业务方品牌（如 BDBGAME），不强制显示 1goplus 文案。
  • 除品牌文案外，页面结构、视觉层级、交互表现按 1goplus 对齐。

3. 产品表现硬性要求（必须与 1goplus 一模一样）

3.1 错误反馈基线

  • 错误反馈优先使用顶部 Toast + 输入框错误态。
  • 未开放功能入口统一 Toast：Coming Soon。

3.2 一致性验收标准

  • 同功能页面的布局、层级、色彩、间距与 1goplus 对齐。
  • 关键路径（登录/注册/首页/签到/游戏/个人中心）交互顺序与反馈一致。
  • 成功态、失败态、空态、加载态视觉语言一致。
  • 若文档与线上冲突，以 1goplus 实际页面行为为准修正。

4. 执行原则

  • 原则 1：范围可裁剪，表现不可打折。
  • 原则 2：所有“V1不做”入口要可见，并给明确 Toast 反馈。
  • 原则 3：先完成对标一致，再做业务差异。
